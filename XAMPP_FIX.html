🗺️ IExplore Laguna — XAMPP 403 Fix Guide

You're seeing Error 403 Access Forbidden because XAMPP's Apache is not allowing .htaccess overrides for your project folder. Follow the steps below to fix it permanently.

⚠️ Root cause: XAMPP ships with AllowOverride None in its default Apache config. This blocks all .htaccess files from working, which Apache reports as a 403.

Fix 1 — Edit httpd.conf (Required)

1 Open XAMPP Control Panel → click Config next to Apache → choose httpd.conf
File path: C:\xampp\apache\conf\httpd.conf
2 Press Ctrl+F and search for: htdocs

Find this block (around line 250): 
<Directory "C:/xampp/htdocs">
    Options Indexes FollowSymLinks Includes ExecCGI
    AllowOverride None
    Require all granted
</Directory>
Change AllowOverride None → AllowOverride All: 
<Directory "C:/xampp/htdocs">
    Options Indexes FollowSymLinks Includes ExecCGI
    AllowOverride All
    Require all granted
</Directory>
3 Save the file, then in XAMPP Control Panel click Stop → Start on Apache to restart it.

Fix 2 — Verify mod_rewrite is enabled (if still 403)

4 In the same httpd.conf, search for: 
#LoadModule rewrite_module modules/mod_rewrite.so
Remove the # at the start so it reads: 
LoadModule rewrite_module modules/mod_rewrite.so
Restart Apache again.

Fix 3 — Check project folder permissions (Mac/Linux XAMPP)

5 On macOS/Linux only — open Terminal and run: 
chmod -R 755 /Applications/XAMPP/xamppfiles/htdocs/iexplore-laguna
On Windows this is not needed.

Test your fix

After restarting Apache, visit these URLs in order:

  1. http://localhost/iexplore-laguna/test_db.php — should show green ✅ for all tables
  2. http://localhost/iexplore-laguna/ — should show the homepage
✅ If test_db.php works but index.php still shows 403 — the issue is specifically with .htaccess. Repeat Fix 1 and make sure you saved the file.
⚠️ If you get a blank white page — there's a PHP error. Enable error display temporarily: open includes/config.php and make sure DEBUG_MODE is set to true.
